JSX 如何在浏览器运行

Jun 20, 2016

今天第一次看React，接触了 JSX 这门『语言』。实际上，JSX 是 JavaScript 的语法扩展（sytax sugar），JS 引擎不会直接运行 JSX 代码，一般情况下，用 JSX 写的代码需要通过 web 打包工具编译成基本 JS 代码后才能运行，而为了提高效率，这种编译都是在服务端做的。

  1. 去掉 browser 库的引用
  2. 在这部分代码中加入输出 log 的语句
  3. 尝试用浏览器运行

结果发现这部分 log 是不会执行的，茅塞顿开啊！原来浏览器 JS 引擎是不会执行 type 为 text/babel的脚本，这就是为什么这部分脚本执行不会报错的原因。

并且，通过 browser.min.js 源码， 可以知道，这个编译器会遍历 <script> 标签，找到所有 type 是 text/babel 的脚本，将其翻译成对应的 JS 代码，然后执行。
